【代表プロジェクト】
プロジェクト名: MBTI × 生成AI 体験アプリ
- 概要: ユーザーの回答から MBTI タイプを推定し、AI が音楽/写真/ストーリーなどの生成体験を提供する Web アプリ。
- 役割: フルスタック（設計、実装、プロンプト設計、運用）
- 技術: Ruby on Rails, OpenAI API, Docker, PostgreSQL, Hotwire/Stimulus, SCSS
- 主要機能:
  - MBTI 診断（`app/models/mbti_question.rb`, `app/models/mbti_result.rb`）
  - AI 生成サービス層（`app/services/ai_*_service.rb`, `app/services/openai_service.rb`）
  - ストーリーモード/セッション管理（`app/models/mbti_session.rb`, `db/migrate/*mbti_sessions.rb`）
  - 画面: 診断/結果/AI 結果/モード選択（`app/views/mbti/*.html.erb`）
  - 例外時フォールバック（`app/services/openai_fallbacks.rb`）とパーサ（`app/services/openai_parsers.rb`）
  - アセット/スタイル整備（`app/assets/stylesheets/*.scss`）
  - ルーティング/コントローラ（`config/routes.rb`, `app/controllers/mbti_controller.rb`）
 - 工夫点:
  - OpenAI 呼び出しの共通化とフォールバックで可用性を確保
  - プロンプトを用途ごとに分離（音楽/写真/ストーリー）し保守性を向上（`openai_prompts.rb`）
  - セッションテーブルでモード/状態を管理し、体験の継続性を担保
  - UI は ERB + SCSS で軽量に、体験部分は Stimulus で最小限の動的化

【技術詳細（抜粋）】
- サービス層設計:
  - `OpenAIService`: API クライアントの単一責務化、レート制御/エラーハンドリング
  - `AIGameMasterService` / `AIMusicService` / `AIPhotoService`: ドメインごとにプロンプト/パースを分離
  - `OpenAIFallbacks`: API 失敗時に返すダミー/テンプレート応答で UX を維持
  - `OpenAIParsers`: 生成テキストの構造化（JSON ライク/タグ抽出）
- データモデル:
  - `MBTISession`: ストーリーモード、状態、カスタムストーリーを持つ
  - `MBTIResult`: MBTI タイプ、説明、推奨コンテンツ紐付け
- テスト/品質:
  - Rails 標準テストを活用（`test/models/mbti_session_test.rb` 他）
  - ログ/例外通知（本番環境想定の設定は `config/environments/production.rb`）
